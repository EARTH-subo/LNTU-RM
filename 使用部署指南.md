## 1. 项目能否直接运行？

**不能。** 收到项目源码后，需要完成以下准备工作才能运行：

| 缺失项 | 说明 |
|--------|------|
| Python 解释器 | 需自行安装 Python 3.10+ |
| PyTorch | 需根据 GPU 型号选择对应版本安装 |
| 项目依赖包 | 需通过 `pip install -r requirements.txt` 安装 |
| SAM3 模型权重 | 需单独下载 `sam3.pt` 文件（约 2GB+） |
| 模型路径配置 | 需在 `main.py` 第 53 行修改为你电脑上的实际路径 |

## 2. 硬件与系统要求

### 最低配置

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11（64 位） |
| Python | 3.10 或 3.11 |
| GPU | NVIDIA 独立显卡，显存 ≥ 6GB（推荐 ≥ 8GB） |
| CUDA | 11.8 或 12.1 |
| 内存 | ≥ 16GB |
| 硬盘 | ≥ 10GB 可用空间（含模型权重） |

### 关于 GPU 品牌的说明

| GPU 品牌 | 是否支持 | 说明 |
|----------|---------|------|
| **NVIDIA** | ✅ 完全支持 | 本项目基于 CUDA 开发，NVIDIA 显卡是唯一官方支持的 GPU |
| **AMD** | ❌ 不支持 | AMD 显卡在 Windows 上不支持 CUDA，当前代码无法使用 SAM3 功能 |
| **Intel** | ❌ 不支持 | Intel 核显/独显不支持 CUDA |
| **无独显（纯 CPU）** | ⚠️ 仅手动标注 | 可运行程序界面和手动标注，但 SAM3 智能辅助功能不可用（代码中硬编码了 CUDA 调用） |

> **总结：要使用 SAM3 智能标注功能，必须拥有 NVIDIA 独立显卡。** 没有 NVIDIA 显卡的电脑只能使用手动标注功能（需修改代码适配 CPU，见第 10 节说明）。

## 3. 第一步：安装 Python

### 3.1 下载 Python

前往 Python 官网下载：[https://www.python.org/downloads/](https://www.python.org/downloads/)

推荐版本：**Python 3.11.x**（项目测试环境为 3.11）

### 3.2 安装时的重要选项

运行安装程序时，**务必勾选**：

```
☑ Add Python 3.11 to PATH
```

然后点击 **Install Now** 完成安装。

### 3.3 验证安装

打开命令提示符（按 `Win + R`，输入 `cmd`，回车），输入：

```bash
python --version
```

应输出类似 `Python 3.11.x`，即安装成功。

## 4. 第二步：确认 GPU 类型与 CUDA 版本

### 4.1 确认是否拥有 NVIDIA 显卡

按 `Win + R`，输入 `cmd`，回车，输入：

```bash
nvidia-smi
```

- **如果输出了显卡信息表格**：说明你有 NVIDIA 显卡且驱动已安装。在表格右上角找到 **CUDA Version**，记下这个数字（如 `11.8`、`12.1`、`12.4` 等）。
- **如果提示"不是内部或外部命令"**：说明没有 NVIDIA 显卡或未安装驱动。请前往 [NVIDIA 驱动下载页](https://www.nvidia.cn/Download/index.aspx) 安装最新驱动后重试。
- **如果确认没有 NVIDIA 显卡**：请参考第 10 节"不同硬件场景说明"。

### 4.2 CUDA 版本对照表

| nvidia-smi 显示的 CUDA Version | 可选的 PyTorch CUDA 版本 |
|-------------------------------|------------------------|
| 11.8 | cu118 |
| 12.1 | cu121 |
| 12.2 | cu121 |
| 12.3 | cu121 |
| 12.4 及以上 | cu124 或 cu121 |

> **核心规则：PyTorch 的 CUDA 版本号必须 ≤ nvidia-smi 显示的 CUDA Version。**

## 5. 第三步：安装 PyTorch

PyTorch 必须在项目依赖之前单独安装，因为需要根据 GPU 选择对应版本。

### 场景 A：NVIDIA 显卡 + CUDA 11.8

```bash
pip install torch==2.5.0 torchvision==0.20.0 torchaudio==2.5.0 -f https://mirrors.aliyun.com/pytorch-wheels/cu118
```

### 场景 B：NVIDIA 显卡 + CUDA 12.1

```bash
pip install torch==2.5.0 torchvision==0.20.0 torchaudio==2.5.0 -f https://mirrors.aliyun.com/pytorch-wheels/cu121
```

### 场景 C：NVIDIA 显卡 + CUDA 12.4 及以上

```bash
pip install torch==2.5.0 torchvision==0.20.0 torchaudio==2.5.0 -f https://mirrors.aliyun.com/pytorch-wheels/cu124
```

> 如果阿里云镜像源没有对应版本，可前往 PyTorch 官网获取最新安装命令：[https://pytorch.org/get-started/previous-versions/](https://pytorch.org/get-started/previous-versions/)

### 场景 D：无 NVIDIA 显卡（仅 CPU）

```bash
pip install torch==2.5.0 torchvision==0.20.0 torchaudio==2.5.0 --index-url https://download.pytorch.org/whl/cpu
```

> ⚠️ CPU 版本仅能运行程序界面和手动标注，SAM3 智能辅助功能不可用。

### 验证 PyTorch 安装

```bash
python -c "import torch; print(f'PyTorch: {torch.__version__}'); print(f'CUDA可用: {torch.cuda.is_available()}')"
```

- NVIDIA 显卡用户应看到 `CUDA可用: True`
- CPU 用户应看到 `CUDA可用: False`（正常，但 SAM3 不可用）

## 6. 第四步：创建虚拟环境并安装项目依赖

### 6.1 创建虚拟环境（推荐）

在项目根目录下打开命令提示符，执行：

```bash
python -m venv myenv
```

激活虚拟环境：

```bash
myenv\Scripts\activate
```

> 激活成功后，命令行前面会出现 `(myenv)` 标识。

### 6.2 安装项目依赖

确保虚拟环境已激活，然后执行：

```bash
pip install -r requirements.txt
```

> **注意**：`requirements.txt` 中包含 `git+https://github.com/facebookresearch/sam3.git`，安装时需要联网访问 GitHub。如果网络不稳定，可能需要多次尝试或使用代理。

### 6.3 如果 requirements.txt 安装失败

可以分步安装核心依赖：

```bash
pip install pyside6==6.4.2 numpy opencv-python pillow einops pycocotools scipy tqdm iopath matplotlib timm ftfy psutil torchmetrics omegaconf numba huggingface-hub pandas scikit-learn setuptools
pip install git+https://github.com/facebookresearch/sam3.git
```

## 7. 第五步：下载 SAM3 模型权重

### 7.1 下载地址

从 HuggingFace 下载 `sam3.pt` 权重文件：

[https://huggingface.co/facebook/sam3/tree/main](https://huggingface.co/facebook/sam3/tree/main/)

找到 `sam3.pt` 文件，点击下载。文件较大（约 2GB+），请耐心等待。

### 7.2 放置位置

将下载的 `sam3.pt` 文件放到项目根目录下：

```
D:\LuoHuaLabel-main\sam3.pt
```

> 如果你的项目放在其他路径，请放到对应的项目根目录下。

---

## 8. 第六步：配置模型路径

用文本编辑器（如记事本、VS Code）打开项目中的 `main.py` 文件，找到第 53 行：

```python
self.sam_client.load_model_async(r"D:\LuoHuaLabel-main\sam3.pt")  # 模型路径
```

将路径修改为你电脑上 `sam3.pt` 的实际绝对路径。例如：

- 如果 sam3.pt 放在项目根目录下：`r"D:\LuoHuaLabel-main\sam3.pt"`
- 如果放在其他位置：`r"E:\Models\sam3.pt"`

> **路径前面的 `r` 不能删掉**，它表示原始字符串，防止反斜杠被转义。

## 9. 第七步：启动程序

### 9.1 激活虚拟环境

```bash
cd D:\LuoHuaLabel-main
myenv\Scripts\activate
```

### 9.2 运行主程序

```bash
python main.py
```

### 9.3 启动后的状态

- 程序窗口会立即弹出，此时可以正常使用手动标注功能
- SAM3 模型在后台异步加载，状态栏会显示"正在后台加载模型，请稍候..."
- 加载完成后状态栏变为绿色"模型加载成功"，此时 SAM 智能辅助功能可用
- 模型加载通常需要 10~30 秒，取决于显卡性能

## 10. 不同硬件场景说明

### 场景一：NVIDIA 显卡（推荐，完全支持）

这是唯一能完整使用所有功能的场景。按照上述步骤 1~9 正常操作即可。

**推荐显卡**：RTX 3060 及以上（显存 ≥ 8GB）

### 场景二：AMD 显卡（不支持 SAM3）

当前代码中 SAM3 推理硬编码了 CUDA 调用（`model.to("cuda")`、`torch.autocast(device_type="cuda")`），AMD 显卡在 Windows 上无法使用 CUDA。

**可用的替代方案**：
- 在 Linux 系统上通过 ROCm 运行 PyTorch（需要自行适配代码中的 CUDA 调用为 ROCm）
- 仅使用手动标注功能（需安装 CPU 版 PyTorch，SAM3 功能不可用）

### 场景三：Intel 核显/独显（不支持 SAM3）

与 AMD 情况相同，Intel GPU 不支持 CUDA。

**可用方案**：安装 CPU 版 PyTorch，仅使用手动标注功能。

### 场景四：无独显，纯 CPU（仅手动标注）

安装 CPU 版 PyTorch 后可以启动程序界面，但 SAM3 智能辅助功能不可用。

**具体操作**：
1. 按第 5 步场景 D 安装 CPU 版 PyTorch
2. 按第 6 步安装其他依赖
3. 无需下载 sam3.pt 模型
4. 启动程序后，状态栏会显示模型加载失败（可忽略）
5. 手动标注功能（矩形、多边形、点、旋转框）正常使用
6. SAM 智能辅助开关和提示词功能不可用

## 11. 常见问题排查

### Q1：运行 `python main.py` 报错 `ModuleNotFoundError: No module named 'cv2'`

**原因**：未安装项目依赖。

**解决**：

```bash
pip install opencv-python
```

或完整安装所有依赖：

```bash
pip install -r requirements.txt
```

### Q2：运行 `python main.py` 报错 `ModuleNotFoundError: No module named 'torch'`

**原因**：未安装 PyTorch。

**解决**：按照第 5 步安装对应版本的 PyTorch。

### Q3：PyTorch 安装后 `torch.cuda.is_available()` 返回 `False`

**可能原因与解决**：

| 原因 | 解决方法 |
|------|---------|
| 安装了 CPU 版 PyTorch | 卸载后重新安装 GPU 版：`pip uninstall torch torchvision torchaudio`，然后按第 5 步重装 |
| PyTorch CUDA 版本与驱动不匹配 | 检查 `nvidia-smi` 的 CUDA Version，安装对应或更低版本的 PyTorch |
| NVIDIA 驱动未安装或版本过旧 | 前往 [NVIDIA 驱动下载页](https://www.nvidia.cn/Download/index.aspx) 更新驱动 |

### Q4：启动后状态栏显示模型加载失败

**可能原因**：

1. `sam3.pt` 文件路径配置错误 — 检查 `main.py` 第 53 行的路径是否正确
2. `sam3.pt` 文件损坏 — 重新下载
3. 显存不足 — SAM3 需要至少 6GB 显存，关闭其他占用显存的程序
4. CUDA 不可用 — 确认 `torch.cuda.is_available()` 返回 `True`

### Q5：安装 `git+https://github.com/facebookresearch/sam3.git` 失败

**原因**：网络问题，无法访问 GitHub。

**解决**：

1. 使用代理或 VPN
2. 多次重试
3. 手动克隆 SAM3 仓库后本地安装：

```bash
git clone https://github.com/facebookresearch/sam3.git
cd sam3
pip install -e .
```

### Q6：程序启动后界面卡顿或假死

**可能原因**：SAM3 模型加载或推理占用大量显存/CPU。

**解决**：
- 等待模型加载完成（状态栏变绿后恢复流畅）
- 关闭其他占用 GPU 的程序
- 降低图片分辨率

### Q7：提示词功能输入后无反应

**可能原因**：

1. 模型尚未加载完成 — 等待状态栏显示"模型加载成功"
2. 当前为点标注模式 — 点标注模式下 SAM 不可用，切换到矩形/多边形/旋转框模式
3. 显存不足 — 提示词全图提取对显存要求较高

### Q8：pip 安装速度很慢

**解决**：使用国内镜像源加速：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```
